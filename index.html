维特度的技术栈介绍

关于基础设施架构：
  1. 使用docker容器技术部署我们的应用和组件；
  2. 进程管理工具采用Runit
  3. 对于容器的编排和调度工具采用 Kubernetes；
  4. 对于深度学习中的4大主流框架（Tensorflow、Mxnet、Pytorch、Caffe/Caffe2）已经封装成基础镜像，通过Nvidia Docker 和Kubernetes的插件可以非常方便的在容器中构造一个完善的训练环境或运行环境，这样就可以通过容器技术的轻量化虚拟技术实现环境隔离进程隔离，从而完美解决了裸机下对不同软件、同一软件版本、不同版本驱动等一系列的环境问题，为开发人员和运维人员解决了基础环境的问题，这样就节约了大量的配置环境的时间，只需要关注业务实现即可，因此可以加速产品的迭代效率，为后续的工作赢得更多时间；
  5. Docker仓库采用Vmware的企业级开源产品Harbor；
  6. 关于Kubernetes的部署和升级采用helm和operator进行简化，使用Git仓库作为helm的chart管理能很好的做包的版本控制；
  7. 实现了K8s调度GPU节点和容器内调用GPU计算；
  8. 私有Git仓库采用Gitea搭建；
  9. 实时视频流rtmp、hls服务器采用srs作为服务端，客户端采用深度学习框架+opencv+ffmpeg进行处理，并输出到srs服务器，实现对视频的实时处理；
  10. k8s的性能监控采用普罗米修斯等工具，图形化用grafana；
  11. 日志收集和处理，以及监控和预警 ，采用EFK系统进行处理；（Elasticsearch、Fluentd、Kibana）；
  12. 所有项目的应用程序实现分布式高可用架构（可以选择微服务架构（实现Service Mesh的Istio），通过消息队列实现程序间解耦，protobuf作为序列化，grpc作为调用协议）；
  13. 分布式存储系统采用Ceph，为k8s提供持久化支持与海量存储；
  14. 分布式Redis的集群方案 采用codis 或者pika实现 ；
  15. RDB的方案采用Newsql实现的 cockroach 或tidb；
  16. 通过k8s实现RabbitMQ集群，用于分布式消息队列；
  17. 用户管理采用openLDAP实现对用户的统一管理，避免每套系统重新录入用户的重复工作；
关于代码质量：
  1. Git flow约定代码管理流程；
  2. 采用Gerrit做Code Review；
  3. 采用Sonarqube作为代码的静态分析工具；
  4. Jenkins multibranch pipeline作为CI的创建类型，方便对不通分支的代码进行测试和构建，保证不通分支的代码质量；
关于CI/CD的工作：
  1. CI/CD的工具我们采用Jenkins，因其插件多、开源、使用广泛方便；
  2. 采用MulitBrach Pipeline + Jenkinsfile进行构建流水线的配置，这样做的好处就是更加灵活并且支持多分枝构建，根据需要可以通过Groovy定制自己的流程，如果流水线有变动的话只需要改动项目目录下的Jenkinsfile即可触发并且改变相应构建流程，对于流水线的控制，不但快而且可以准确把控；
  3. 对于Pipeline的stage根据需求加入IM通知，由于我们工作流采用Dingtalk，所以就会将Jenkins的构建信息推送到钉钉工作组，方便不通阶段通知不同群组的人，至于为什么不用Slack，BarryChat、Email等，因为Dingtalk足够实现消息通知，不需要再增加额外工具,少即是多。不用Email 是因为没有即时工具及时，而且不能即时与相关人员在线讨论，沟通起来太繁琐，IM工具可以很直观简洁的表达；
  4. 关于CI/CD流程：用户本地git review后提交代码 -> 提交到Gerrit库 -> 执行Sonarqube对代码进行静态分析（从源码就开始注重代码质量） -> 触发Jenkins自动测试, 测试通过标记为Verified -> 人工审核Review，review通过 -> Gerrit执行Replication -> push Git remote -> 触发远程仓库下的Jenkins Pipeline->Build docker image -> Push Docker image to Harbor -> Deploy to test environment for test from Harbor -> Run E2E test -> 将结果通知给当前代码分支的负责人（架构师、测试经理或发布人员等）->管理员将代码merrage到 release 分支-> 进过充分测试和准备后由管理人员合并到Master 分支->触发发布程序（调用K8s的API进行发布，让程序灰度到生产） ；
关于深度学习：
  1. 对于深度学习方面借助k8s通过大数据（spark、storm等工具）和机器学习框架实现分布式训练，加快训练速度，加强通过海量数据训练大模型的能力；
  2. 通过模型转换技术将训练的算法模型部署到嵌入式设备（主要是移动设备arm架构硬件），实现移动设备的智能化；
  3. 嵌入式领域采用FPGA来加速我们的算法模型，MCU采用先进开源的RISC-V指令集进行处理器的设计，在AI和IoT将发挥巨大优势
关于开发语言：
  1. 微服务和高并发场景采用golang或者rust进行编写，因为他们特别适合通过docker实现微服务架构，充分体现灵活性和扩展性以及高性能；
  2. 在Witdo的新产品中将采用RISC-V作为专用硬件的架构，Chisel作为硬件设计语言；
  3. 嵌入式开发首选Rust，因为它更安全在语言层面解决了很多C/C++ 一直存在却很难避免的问题，比如指针问题和运行时的一些问题；
  4. RISC-V + Rust 在Witdo将成最佳组合,Rust和RISC-V都是在没有历史包袱，并且建立在各子行业几十年踩坑经验之上的，避免了很多坑，优化了很多地方，所以既然都是新的开源项目何不在写新的项目时用rust呢?至于以前的c，c++代码，rust可以通过binding去调用，新的核心代码完全可以用rust。
  5. 快速原型或者对性能要求不算高，可以用python进行快速实现；
  6. 在区块连开发上结合需求首选golang进行开发，对性能要求比较的地方改用Rust，尽量避免使用c/C++

了解 我们的产品

维特度致力于在大数据、云计算技术、区块连、人工智能、智能芯片领域的创新和研究。借助大数据、云计算实现深度学习在海量数据上的分布式训练，从而加快训练速度，提升训练大模型的能力。目前我们已经通过Kubernetes和Docker容器技术实现了多机多GPU的分布式训练方案。在成熟的大数据、云计算大环境下，维特度将结合5G通讯、人工智能、区块链等技术提供智能家居、车联网、物联网方案。

DocStack（多客）

DocStack诞生于公司内部的文档管理需求，公司内部定期会组织分享和培训，经常会有会议笔记和相关文档，虽然有很多云笔记可以用 ，但是企业的资料还是希望能够自己掌控，DocStack已经在GitHub开源。了解更多

Deeploy（深邃）

Deeploy为企业提供在线设计深度模型、训练模型、打包模型、部署模型的解决方案，主要目的是解决本地机器学习复杂环境的配置以及分布式多机多GPU训练的痛点，Deeploy可以实现多个环境的隔离，以多租户的方式提供云端机器学习的加速方案和部署方案。

Deepit（深谷）

Deepit是用于IoT领域终端设备的人工智能芯片方案，兼容Tensorflow、Mxnet、Pytorch的算法模型部署。

Nyx（尼克斯）

Nyx是一套基于Istio实现的Service Mesh微服务方案。Nyx已经用于Witdo的多个线上产品中，拥有非常好的弹性伸缩性和稳定性。
